1. Introduction

Your privacy is fundamental to us. This Privacy Policy describes how the GiftRoulette app collects, uses, and protects your personal data in compliance with the General Data Protection Regulation (GDPR) EU 2016/679.

GiftRoulette is a free application that allows you to organize group gift exchanges in a fun and synchronized way. The app uses Google Firebase Firestore for real-time data management and Google AdMob for displaying advertisements that allow us to offer the service for free.

Important: GiftRoulette only collects data strictly necessary for the service to function. We do not collect sensitive personal information and all data is treated with maximum security.

3. Data Collected and Purposes

3.1 Data Stored Locally (on your device)

This data is stored only on your device and is never sent to external servers:

  • Username: The name you choose to identify yourself in sessions
  • Gift list: The gifts you create for exchanges
  • Extraction history: Past extractions saved locally (SQLite database)
  • App preferences: Language, privacy consent, onboarding status
  • Duo codes: Saved couple codes for quick access

Purpose: Provide app functionality without requiring account registration.

Legal basis: Contract performance (app usage).

Retention: Until app uninstallation or manual data deletion from settings.

3.2 Data Stored on Google Firebase Firestore

During group features usage, some data is temporarily saved on Firebase Firestore to enable real-time synchronization:

GiftRoulette Sessions

Data: Session code (8 chars), host device ID, participant count, extraction settings, creation timestamp

Retention: 24 hours (automatic deletion)

Session Participants

Data: Device ID, chosen name, selected gift (label and description), received gift, join timestamp

Retention: 24 hours (automatic deletion with session)

Duo Mode

Data: Duo code, device IDs (2 users), chosen names, wishlists with gift names and optional links, timestamps

Retention: Permanent until manual deletion by users

Offline Users

Data: Offline user name, gift description (entered by host for participants without smartphones)

Retention: 24 hours (deleted with session)

Purpose: Real-time synchronization between participants' devices, managing random extractions, sharing wishlists in Duo mode.

Legal basis: Contract performance (use of app's collaborative features).

Server location: Europe-west1 (Belgium) - GDPR compliance guaranteed.

Firebase Data Security

All data is transmitted via TLS/SSL encrypted connections. Firebase servers comply with Google security standards and are GDPR compliant. Data access is protected by Firestore security rules.

3.3 Device Identifiers

The app uses anonymous device identifiers generated locally to:

  • Uniquely identify participants in sessions without requiring accounts
  • Allow automatic reconnection to active sessions
  • Associate received gifts with the correct device

Technology: UUID v4 generated locally, no link to real identity.

Google Advertising ID: Used only by AdMob for personalized ads (see section 3.4).

3.4 Data Collected by Google AdMob

GiftSwap uses Google AdMob to display advertisements. AdMob may collect:

  • Advertising identifiers: Google Advertising ID (GAID), App Set ID
  • Device information: IP address, device type, model, operating system, language
  • Ad interaction data: Views, clicks, viewing time
  • Approximate demographics: Age range, general interests (with consent only)

Purpose: Ad delivery, ad personalization (with consent), performance measurement, fraud prevention.

Legal basis: Explicit consent (for personalized ads) or legitimate interest (for generic ads).

Consent management: Google Consent Mode v2 implemented, GDPR compliant.

Ad Management

App Settings → Privacy Management → Ad Preferences
Google Account → Ad Settings

4. Data Architecture and Firestore Operation

4.1 Firebase Database Structure

GiftSwap uses two main collections in Firestore:

Collection: "sessions"

Content: Document for each GiftSwap session with code, configurations, status
Subcollection: "participants" with each participant's data
TTL: 24 hours from creation

Collection: "duos"

Content: Document for each Duo pair with code, wishlists of both users
Persistence: Permanent until manual deletion
Access: Only via unique 8-character code

4.2 Real-time Synchronization

When you participate in a session, the app maintains a real-time connection with Firestore to:

  • See new participants joining the session
  • Receive extraction results when the host starts the draw
  • Update wishlists in Duo mode when your partner adds gifts

Implemented optimizations:

  • Batch writes to reduce number of write operations
  • Listeners active only during actual screen usage
  • Automatic disconnection when you exit the session

4.3 Automatic Data Deletion

For GDPR compliance and resource optimization:

GiftSwap Sessions

Deletion: Automatic after 24 hours from creation
Method: Cloud Function scheduled every 6 hours
Includes: Session document + all participants

Duo Mode

Deletion: Manual only by users
Method: "Leave Duo" button in app
Effect: Immediate document deletion

Important: GiftRoulette sessions are automatically deleted after 24 hours even if the session is still active. Make sure to complete the extraction and save any important information locally (history is automatically saved on your device).

11. Automated Decisions and Profiling

GiftSwap does NOT use automated decision-making that produces legal or significant effects on users.

Extraction Algorithm

The only automated process is the random draw algorithm for gift assignment:

  • Purpose: Randomly assign a gift to each participant
  • Logic: Random shuffle ensuring no one receives their own gift
  • Transparency: Completely random process without favoritism
  • Consequences: No legal effect, only recreational/social

AdMob Profiling

Google AdMob may perform advertising profiling only with your explicit consent:

  • Interest analysis based on online behavior
  • Creation of advertising profiles for ad targeting
  • Automated decisions without human intervention

Control: You can always refuse or withdraw consent by choosing "Non-personalized ads" in app settings or via Google Ad Settings.
